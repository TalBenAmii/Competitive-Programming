1. תחזוקה של set ואיטרטור עבור האיבר הבא ל GET. 
נכניס איברים (בקשות ADD) כל עוד כמות האיברים קטנה מבקשת ה GET הבאה.
נדאג לעדכן את האיטרטור בהתאם כלומר אם האיבר שהכנסנו קטן מהאיבר הנוכחי (או שאיטרטור מצביע לסוף ה SET) אליו מצביעים (ולא היו עוד בקשות GET) נקטין אותו, אחרת נשאיר אותו במקומו.
2. נכונות נובעת מכך ש SET דואג לשמירת סדר מיון האיברים.
3. time: O((N+M)*K*logM)
space: O(M)
N - gets requests, M- add requests (total elements in set), K- data sets
